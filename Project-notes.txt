DataFlow

User uploads file
      ↓
Flask parses file (npm/pypi/maven parser)
      ↓
Resolver fetches full dep tree (direct + transitive)
      ↓
CVE clients query NVD + OSV in parallel
      ↓
Flask returns: dep graph + mediation + CVEs
      ↓
React renders report + graph

Critical Gaps (should fix before launch)
1. No rate limiting on the backend
Anyone can spam /api/analyze repeatedly, hammering NVD/OSV APIs and getting your key banned. Need a simple request limiter.
2. CORS is wide open
pythonCORS(app)  # allows ALL origins
In production anyone can call your API from any website. Should restrict to your frontend URL only.
3. No input size limit
User could upload a 50MB pom.xml and crash the server. Need max file size check.
4. package-lock.json not supported
As we just discussed — most accurate source of truth for npm projects. Currently ignored.

🟡 Important Gaps (affects user experience)
5. Scan results not saved
User scans, navigates away, results are gone forever. No history, no way to compare scans over time.
6. No export
User gets 275 vulnerabilities — no way to download as PDF or CSV to share with their team or manager.
7. Search page doesn't know ecosystem context
If user scanned a Maven project then goes to search, the search doesn't know to query Maven ecosystem — it defaults to npm.
8. CVE deduplication
Same CVE can appear multiple times if a package appears in multiple paths (e.g. lodash as direct AND transitive). Currently shown twice.
9. No timeout feedback
If OSV/NVD is slow, user sees loading forever with no indication of what's happening server-side.

🟢 Nice to Have (future improvements)
10. package-lock.json / poetry.lock / gradle.lock support
Lock files give exact installed versions — much more accurate than package.json alone.
11. Fix verification
After user upgrades a package, they should be able to re-scan and see "✅ CVE resolved" confirmation.
12. Severity trend
"Last scan had 5 CRITICAL, this scan has 3 CRITICAL — you fixed 2" — scan comparison over time.
13. GitHub integration
Scan directly from a repo URL instead of uploading a file.
14. Email alerts
"New CVE found in your lodash version" — notify user when new vulnerabilities are published for their packages.

My priority suggestion:
Fix in this order:

CVE deduplication — affects accuracy right now

Export (PDF/CSV) — most requested feature in security tools
package-lock.json — accuracy improvement
Scan history — major usability improvement
